HTML

  • Text elements, image elements, header elements, and list elements are commonly used to add content to pages
  • Meta-related elements, content division elements, and semantic elements are commonly used to divide and describe a page

CSS

  • Inline styles apply to a specific HTML tag, using a style attribute with a CSS rule to style a particular page element only.
  • An internal style sheet contains CSS rules for the webpage in the head section of the HTML file
  • External CSS style sheets allow developers to keep all their CSS rules in a separate file, which makes design changes easier.
  • a margin property indicates how much space we want outside of an element. When we set it to 0, we are indicating we want NO extra space around element.
  • Padding property adds space around the content inside an element. Increase padding, making entire element bigger.
  • The block value assigns each image to start on a new line.

Git

  • A pull request, or PR, is used to merge a comparison branch (in our case, the feature branch) into the base branch.
  • pwd - print working directory
  • ls - list of files in working directory
  • cd - change directory. Example: cd Downloads
  • cd .. - goes back one level
  • mkdir bootcamp - make directory
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • We use the git add -A command to commit all files in the working branch.
  • We use the -m flag to associate a message with our commit.
  • Use git pull origin main to pull in the base branch before pushing up any code. If branches are in sync, should see "Already up to date."
  • git push origin feature/add-html

JavaScript

  • Control flow: the order in which a computer executes code in a JavaScript file.
  • To declare a variable, we use the var keyword
  • Whereas the style.css file is linked within the head element, the script.js file is linked within the body element.
  • We can use conditional statements and loops to control the order in which code is executed, or if it is even executed at all.
  • The strict equality operator (===) checks to see if two values are equal, and returns a Boolean result true if the values are equal and false if the values are not equal.
  • Strings need to be contained within quotation marks.
  • The location of a data item in the array is identified with a unique number called an index.
